학습 내용

1. 서버 내부에 강제적으로 객체수를 하나만 유지시키는 기술
2. 사용 사유
	- 공통 로직인 경우 요청시마다 객체를 생성할 경우 서버 내부 자원의 손실 야기
	- 강제적으로 리소스 절약 측면에서 객체수 조절
	- 로그 기록 기능의 객체들은 client 수와 무관하게 서버 내부에 하나만 존재

3. Singleton Design Pattern
	1. 개발 기술
		1. private 으로 생성자 구현
			-외부에선 절대 객체 생성 불가
		2. public static 메소드 구현
			-객체 제공을 위한 메소드
		3. 객체 생성
			- 자기 자신의 클래스의 멤버 변수로 생성
